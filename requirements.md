

# 要件定義書 兼 開発指示書: 高速・軽量数式処理ライブラリ (Symbolic Engine)

## 1. プロジェクト概要

本プロジェクトは、C++による強力かつ高速な数式処理（CAS: Computer Algebra System）のバックエンドと、Pythonによる直感的なフロントエンドを融合させた、オープンソースの数式処理ライブラリの開発である。
既存のSymPy等のライブラリに比べ、「軽量さ」「C++拡張による高速性」「大学入試・高校数学（数Ⅲレベル）への最適化」を強みとし、GitHubでの公開および開発者コミュニティでの利用を前提とする。

## 2. システムアーキテクチャ

システムは以下の3層構造で構築する。拡張性とメンテナンス性を担保するため、各層の責務を明確に分離する。

1. **Backend (C++):** 抽象構文木（AST: Abstract Syntax Tree）を用いた数式の保持、および簡略化ロジックの実行。
2. **Binding Layer (pybind11):** C++のASTノードおよび計算関数をPythonモジュールとしてバインディング。
3. **Frontend (Python Wrapper):** 人間フレンドリーな文字列入力を解析（パース）し、C++側に渡すインターフェース。Jupyter環境等でのリッチな表示機能。

## 3. 機能要件 (Functional Requirements)

### 3.1. サポートする数式・記号 (高校数学Ⅲレベル)

* **変数・定数:** 任意の英字（`x`, `y`, `a` 等の文字式）、整数、分数
* **主要な定数:** 円周率（`pi` / $\pi$）、ネイピア数（`e`）
* **演算子:** 四則演算（`+`, `-`, `*`, `/`）、べき乗（`^`）
* **関数:** 平方根（`sqrt`）、立方根（`cbrt`）、三角関数（`sin`, `cos`, `tan`）、自然対数（`log` / `ln`）

### 3.2. 構文解析（パーサー）要件

Python側からは文字列（String）で入力を受け付ける。一般のユーザーが数式を直感的に書ける「人間スタイル」をサポートする。

* **スペースの許容:** `x + y` と `x+y` を同一に解釈する。
* **掛け算記号（`*`）の省略:** `5x`, `2pi`, `x sin(x)`, `(x+1)(x-2)` などの表現を自動的に掛け算としてパースする。

### 3.3. 数式の自動簡略化 (大学入試解答レベル)

数式が評価（Evaluate）された際、大学入試で減点されないレベルまで自動で綺麗に整理する。実装は以下の優先順位で段階的に行う。

* **【優先度1】分母の有理化:** * 例: `1 / sqrt(2)` $\rightarrow$ `sqrt(2) / 2`
* **【優先度2】分数・整式の約分と通分・同類項の整理:**
* 例: `1/2 + 1/3` $\rightarrow$ `5/6`
* 例: `2x + 3x` $\rightarrow$ `5x`
* 例: `(x^2 - 1) / (x - 1)` $\rightarrow$ `x + 1`


* **【優先度3】三角関数の特殊角・公式の処理:**
* 例: `sin(pi/4)` $\rightarrow$ `sqrt(2)/2`
* 例: `cos(pi)` $\rightarrow$ `-1`


* **【優先度4】指数・対数の法則・根号の整理:**
* 例: `ln(e^2)` $\rightarrow$ `2`
* 例: `sqrt(12)` $\rightarrow$ `2*sqrt(3)`
* 例: `cbrt(x^3)` $\rightarrow$ `x`



### 3.4. 出力形式

他者への共有やJupyter Notebookでの利用を想定し、出力は **LaTeX形式の文字列** を基本とする。

* 例: `(5*sqrt(2)*pi)/3` $\rightarrow$ `\frac{5\sqrt{2}\pi}{3}`
* Pythonの `__str__` および `_repr_latex_` をオーバーロードし、環境に応じて自動で綺麗な数式が描画されるようにする。

### 3.5. エラーハンドリング

不正な数式が入力された場合（カッコの閉じ忘れ、未定義の関数など）、処理を曖昧にせず、即座にPythonの例外（`ValueError` 等）を発生させる。
他の人が使った際に原因がすぐ特定できるよう、**エラーメッセージは明確な日本語**で出力する。

* 例: `「エラー: カッコが正しく閉じられていません。」`
* 例: `「エラー: 未定義の変数または関数 'invalid_func' が検出されました。」`

## 4. 非機能要件 (Non-Functional Requirements)

* **ユーザビリティ（使いやすさ）:** `import` して一行でパース・計算ができる極めてシンプルなAPI設計にする。
* **ポータビリティ:** Windows, macOS, Linuxの主要3プラットフォームでコンパイル・動作可能とする。
* **コードの堅牢性:** 計算ロジックのバグを防ぐため、主要な簡略化パターンに対するユニットテスト（Pythonの `unittest` または `pytest`）を網羅する。

```text
    └── test_core.py

```

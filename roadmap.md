# SymBuna 開発ロードマップ

AIエージェントは、以下のフェーズを1つずつ順番に実装してください。
前のフェーズが手元で完全に動作し、テストをパスするまで次のフェーズに進んではいけません。

## フェーズ1: 最小構成の構築（MVP）
- [ ] C++側で、変数（x）と整数（int）を保持できる最小限のAST（抽象構文木）クラスを設計。
- [ ] 基礎的なLaTeX出力機能（`to_string()`）の実装。
- [ ] `pybind11` を使ったPythonバインディングの設定（`setup.py` の作成）。
- [ ] Python側から `import symbuna` して、手動で組み立てた木構造からLaTeXが出力できることを確認。

## フェーズ2: 演算子のサポートと基礎的な簡略化（優先度A・B）
- [ ] C++側で四則演算（`+`, `-`, `*`, `/`）とべき乗（`^`）のノードを追加。
- [ ] 分数（有理数）を小数の近似にせず、分子・分母のペアで保持する仕組みの実装。
- [ ] 【優先度B】分数の約分・通分、同類項のまとめる処理（`2x + 3x -> 5x`）の実装。
- [ ] 【優先度A】平方根（`sqrt`）の実装と、分母の有理化（`1/sqrt(2) -> sqrt(2)/2`）の実装。

## フェーズ3: 数Ⅲレベルの記号・関数の拡張（優先度C・D）
- [ ] 円周率（`pi`）、ネイピア数（`e`）、立方根（`cbrt`）のサポート。
- [ ] 三角関数（`sin`, `cos`, `tan`）および自然対数（`log`）のノード追加。
- [ ] 【優先度C】三角関数の特殊角の処理（`sin(pi/4) -> sqrt(2)/2`）の実装。
- [ ] 【優先度D】指数・対数の法則（`ln(e^2) -> 2`）の実装。

## フェーズ4: 人間スタイルパーサーの合体（Python側）
- [ ] Python側で文字列入力を受け取るパーサーを実装。
- [ ] スペースの許容、掛け算記号の省略（`5x`, `2pi`）をパースして、フェーズ3までのC++ ASTオブジェクトに変換するロジックの実装。
- [ ] 日本語でのエラーハンドリング（構文エラーの例外発生）の実装。

## フェーズ5: 仕上げとドキュメント化
- [ ] `pytest` 等を用いた包括的なユニットテストの作成。
- [ ] Jupyter Notebookでリッチに表示するための `_repr_latex_` の実装。
- [ ] 他の人が使えるような丁寧な `README.md` の完成。
